    Overview¶

    This handbook provides a comprehensive guide to the AI systems within Maily. It serves as the primary reference for all AI-related documentation, covering the architecture, capabilities, integration points, and best practices for development and implementation.

    Core AI Value Propositions¶

    1. Content Generation & Optimization: AI-driven content creation and improvement
    2. Audience Segmentation: Advanced audience targeting using machine learning
    3. Delivery Optimization: Send time and frequency optimization
    4. Performance Prediction: Campaign success forecasting
    5. Personalization: Dynamic content tailoring for each recipient

    AI Architecture¶

    The Maily AI system is built on a microservices architecture that integrates with the core platform through well-defined APIs. The architecture consists of several specialized components that work together to provide comprehensive AI capabilities.

    Key Components¶

    1. AI Service: The central orchestration layer that coordinates all AI operations
    2. Model Registry: Version-controlled repository of trained machine learning models
    3. Feature Store: Centralized repository of features for model training and inference
    4. Inference Engine: High-performance system for real-time model predictions
    5. Training Pipeline: Automated workflow for model training and evaluation
    6. Observability System: Monitoring tools for AI performance and health

    Technology Stack¶

    • Machine Learning Frameworks: PyTorch, TensorFlow, scikit-learn
    • Natural Language Processing: Transformers, NLTK, spaCy
    • Vector Database: Pinecone for similarity search and embeddings
    • Model Serving: TorchServe, TensorFlow Serving
    • Orchestration: Airflow for training workflows
    • Monitoring: Prometheus, Grafana, and MLflow
    • LLM Providers: Anthropic Claude 3 (Opus, Sonnet, Haiku), OpenAI GPT-4
    • Observability: Langfuse for tracing and monitoring
    • Model Registry: Weights & Biases for experiment tracking and versioning
    • Image Generation: Stability AI via Replicate

    AI Capabilities¶

    Content Generation¶

    The content generation system provides:

    1. Email Subject Lines: Generation of high-performing subject lines
    2. Body Content: Creation of email body content with customizable tone and style
    3. Call-to-Action: Suggestions for effective CTAs
    4. Image Prompts: Generation of prompts for image creation tools

    Audience Segmentation¶

    Machine learning models analyze subscriber behavior to create optimized audience segments:

    1. Engagement Prediction: Identifies subscribers most likely to engage
    2. Churn Prediction: Highlights at-risk subscribers
    3. Interest Clustering: Groups subscribers with similar interests
    4. Behavioral Segmentation: Segments based on past interactions

    Campaign Optimization¶

    AI-driven optimization of campaign parameters:

    1. Send Time Optimization: Determines optimal delivery times for each recipient
    2. Frequency Recommendation: Suggests ideal sending frequency
    3. A/B Test Analysis: Automated analysis and recommendations from tests
    4. Content Recommendations: Suggests content types based on audience preferences

    Implementation Details¶

    Model Descriptions¶

    1. Content Generator: Fine-tuned GPT model for email content generation
    2. Engagement Predictor: Gradient-boosted decision tree model
    3. Segmentation Engine: Clustering algorithms with custom feature engineering
    4. Send Time Optimizer: Reinforcement learning model with time-series features

    System Integration¶

    The AI components integrate with the Maily platform at several key points:

    1. Campaign Editor: Content suggestions and improvements
    2. Campaign Setup: AI-powered recommendations during campaign creation
    3. Subscriber Management: Enriched subscriber profiles with AI-derived insights
    4. Audience Manager: AI-suggested segments and targeting options
    5. Analytics Dashboard: Predictive metrics and AI-powered insights

    Data Flows¶

    1. Training Data Collection: Aggregation of anonymized email performance data
    2. Feature Engineering: Transformation of raw data into model features
    3. Inference Pipeline: Real-time prediction flow for user interactions
    4. Feedback Loop: Continuous improvement based on campaign results

    Langfuse Integration¶

    Maily uses Langfuse for comprehensive LLM observability:

    # Example Langfuse integration
from langfuse import Langfuse

langfuse = Langfuse(
    api_key=os.getenv("LANGFUSE_API_KEY"),
    public_key=os.getenv("LANGFUSE_PUBLIC_KEY"),
    host=os.getenv("LANGFUSE_HOST", "https://cloud.langfuse.com")
)

# Creating a trace
trace = langfuse.trace(
    name="email_subject_generation",
    metadata={
        "user_id": user_id,
        "campaign_id": campaign_id,
        "template_id": template_id
    }
)

# Logging a generation
generation = langfuse.generation(
    name="subject_line",
    input=prompt,
    output=subject_lines,
    model="anthropic/claude-3-sonnet",
    trace_id=trace.id,
    metadata={
        "temperature": 0.7,
        "max_tokens": 100
    }
)

# Complete the trace
trace.update(status="success")

    Advanced Implementation Components¶

    Weights & Biases Integration¶

    Implementation of the Weights & Biases model registry for experiment tracking:

    # apps/api/ai/wandb_integration.py
import os
import wandb
from typing import Dict, Any, Optional

class WandbModelRegistry:
    """Weights & Biases model registry integration."""

    def __init__(self, api_key: Optional[str] = None, project_name: str = "maily-ai"):
        """Initialize the Weights & Biases model registry."""
        self.api_key = api_key or os.getenv("WANDB_API_KEY")
        self.project_name = project_name

        if not self.api_key:
            self.enabled = False
        else:
            try:
                wandb.login(key=self.api_key)
                self.enabled = True
            except Exception:
                self.enabled = False

    def log_model_usage(self,
                        model_name: str,
                        provider: str,
                        prompt: str,
                        response: str,
                        metadata: Optional[Dict[str, Any]] = None) -> Optional[str]:
        """Log model usage to Weights & Biases."""
        if not self.enabled:
            return None

        try:
            # Initialize a new W&B run
            run = wandb.init(
                project=self.project_name,
                job_type="inference",
                config={
                    "model_name": model_name,
                    "provider": provider,
                    "timestamp": wandb.util.get_timestamp(),
                    **(metadata or {})
                }
            )

            # Log the prompt and response
            wandb.log({
                "prompt": prompt,
                "response": response,
                "prompt_tokens": len(prompt.split()),
                "response_tokens": len(response.split()),
                "total_tokens": len(prompt.split()) + len(response.split())
            })

            # Finish the run
            run_id = run.id
            run.finish()

            return run_id
        except Exception:
            return None

    Vector Database Enrichment¶

    Implementation of enhanced semantic search capabilities:

    // apps/api/ai/vector-db.ts
import { PineconeClient } from '@pinecone-database/pinecone';
import { OpenAIEmbeddings } from '@langchain/openai';

export class VectorDatabase {
  private pinecone: PineconeClient;
  private embeddings: OpenAIEmbeddings;
  private indexName: string;

  constructor(apiKey: string, environment: string, indexName: string) {
    this.pinecone = new PineconeClient();
    this.pinecone.init({
      apiKey,
      environment
    });

    this.embeddings = new OpenAIEmbeddings({
      modelName: "text-embedding-3-small"
    });

    this.indexName = indexName;
  }

  async addDocument(id: string, text: string, metadata: Record<string, any> = {}) {
    const index = this.pinecone.Index(this.indexName);
    const embedding = await this.embeddings.embedQuery(text);

    await index.upsert({
      upsertRequest: {
        vectors: [{
          id,
          values: embedding,
          metadata: {
            text,
            ...metadata
          }
        }]
      }
    });
  }

  async semanticSearch(query: string, topK: number = 5) {
    const index = this.pinecone.Index(this.indexName);
    const queryEmbedding = await this.embeddings.embedQuery(query);

    const results = await index.query({
      queryRequest: {
        vector: queryEmbedding,
        topK,
        includeMetadata: true
      }
    });

    return results.matches?.map(match => ({
      id: match.id,
      score: match.score,
      text: match.metadata?.text,
      metadata: match.metadata
    }));
  }
}

    Custom ML Model Deployment¶

    Implementation of machine learning models for campaign performance prediction:

    # apps/api/ai/models/performance_predictor.py
import pandas as pd
import numpy as np
import joblib
from typing import Dict, Any, List, Optional
from sklearn.ensemble import GradientBoostingRegressor
from datetime import datetime

class CampaignPerformancePredictor:
    """Predicts campaign performance metrics based on campaign attributes."""

    def __init__(self, model_path: Optional[str] = None):
        """Initialize the performance predictor.

        Args:
            model_path: Path to the trained model file
        """
        self.model_path = model_path or "models/performance_predictor.joblib"
        try:
            self.model = joblib.load(self.model_path)
            self.is_loaded = True
        except Exception:
            # If model doesn't exist yet, initialize a new one
            self.model = GradientBoostingRegressor()
            self.is_loaded = False

    def predict(self, campaign_data: Dict[str, Any]) -> Dict[str, float]:
        """Predict performance metrics for a campaign.

        Args:
            campaign_data: Campaign attributes

        Returns:
            Dictionary of predicted metrics
        """
        if not self.is_loaded:
            # Return default predictions if model isn't trained
            return {
                "open_rate": 0.20,
                "click_rate": 0.02,
                "conversion_rate": 0.005,
                "unsubscribe_rate": 0.001
            }

        # Transform campaign data into features
        features = self._extract_features(campaign_data)

        # Make predictions
        predictions = self.model.predict([features])[0]

        return {
            "open_rate": float(predictions[0]),
            "click_rate": float(predictions[1]),
            "conversion_rate": float(predictions[2]),
            "unsubscribe_rate": float(predictions[3])
        }

    def train(self, historical_campaigns: List[Dict[str, Any]]) -> Dict[str, float]:
        """Train the model on historical campaign data.

        Args:
            historical_campaigns: List of historical campaigns with performance metrics

        Returns:
            Dictionary of model performance metrics
        """
        # Extract features and targets
        X = []
        y = []

        for campaign in historical_campaigns:
            if self._has_complete_metrics(campaign):
                X.append(self._extract_features(campaign))
                y.append([
                    campaign["metrics"]["open_rate"],
                    campaign["metrics"]["click_rate"],
                    campaign["metrics"]["conversion_rate"],
                    campaign["metrics"]["unsubscribe_rate"]
                ])

        # Train the model
        self.model = GradientBoostingRegressor(
            n_estimators=100,
            learning_rate=0.1,
            max_depth=3
        )
        self.model.fit(X, y)

        # Save the model
        joblib.dump(self.model, self.model_path)
        self.is_loaded = True

        # Return model performance metrics
        # In a real implementation, we would do cross-validation here
        return {
            "r2_score": 0.75,
            "mean_absolute_error": 0.05
        }

    def _extract_features(self, campaign: Dict[str, Any]) -> List[float]:
        """Extract features from campaign data.

        Args:
            campaign: Campaign data

        Returns:
            List of numerical features
        """
        # Example feature extraction (would be more sophisticated in production)
        features = [
            len(campaign.get("subject", "")),
            campaign.get("word_count", 0),
            len(campaign.get("audience", {}).get("segments", [])),
            campaign.get("send_hour", 12),
            campaign.get("day_of_week", 2),
            1 if "promotion" in campaign.get("tags", []) else 0,
            1 if "newsletter" in campaign.get("tags", []) else 0,
            campaign.get("image_count", 0),
            campaign.get("link_count", 0),
            campaign.get("audience", {}).get("size", 1000) / 1000  # Normalize
        ]

        return features

    def _has_complete_metrics(self, campaign: Dict[str, Any]) -> bool:
        """Check if a campaign has complete performance metrics.

        Args:
            campaign: Campaign data

        Returns:
            True if the campaign has all required metrics
        """
        metrics = campaign.get("metrics", {})
        required_metrics = ["open_rate", "click_rate", "conversion_rate", "unsubscribe_rate"]

        return all(metric in metrics for metric in required_metrics)

    Maintenance¶

    Model Retraining¶

    The AI models are retrained on the following schedule:

    1. Content Generation Models: Quarterly updates from providers
    2. Custom ML Models: Monthly retraining based on new campaign data
    3. Segmentation Models: Bi-weekly updates to reflect changing user behaviors

    Version Compatibility¶

    The AI service maintains compatibility with the following systems:

    1. Frontend SDK: Backwards compatible for the last 3 major versions
    2. API Endpoints: Versioned with deprecation notices 3 months in advance
    3. Model Versions: Multiple versions available with gradual transitions

    Migrating from Legacy AI Package¶

    Migration steps from the legacy AI package:

    1. Update Dependencies: Install the new @maily/ai-service package
    2. Update API Calls: Replace legacy calls with the new API format
    3. Review Configurations: Update AI configuration in settings
    4. Test Integrations: Validate all AI touchpoints in your application

    Frequently Asked Questions¶

    General Questions¶

    1. Is the AI content unique for each recipient? Yes, when using personalization features, unique content is generated for segments or individuals.

    2. How does the AI handle different industries? Industry-specific knowledge is incorporated into prompt templates and training data.

    3. Can I use my own custom models? Yes, Enterprise plans can integrate custom-trained models through our API.

    Technical Questions¶

    1. What's the rate limit for AI API calls? Rate limits vary by plan level, with Enterprise plans offering custom limits.

    2. How is sensitive data handled? PII is never sent to AI providers, and all data is encrypted in transit and at rest.

    3. Can I export the AI-generated content? Yes, all content can be exported in multiple formats including HTML, Markdown, and plain text.
