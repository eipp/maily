    Overview¶

    The Maily frontend is built using a modular package architecture, organized into a monorepo structure using TurboRepo. This approach enables code sharing, consistent development practices, and independent versioning of packages.

    This reference document consolidates information about all frontend packages, providing a comprehensive guide for developers working with the frontend components of the Maily platform.

    Core Packages¶

    UI Component Library¶

    Package: @maily/ui Repository: packages/ui

    The UI Component Library provides reusable, accessible, and consistently styled React components. It serves as the foundation for the entire frontend UI, ensuring a cohesive design experience.

    Key Components¶

    • Layout Components: Container, Grid, Stack, Box
    • Navigation Components: Navbar, Sidebar, Breadcrumbs, Tabs
    • Form Components: Input, Select, Checkbox, RadioGroup, Switch, DatePicker
    • Feedback Components: Alert, Toast, Modal, Spinner
    • Data Display: Table, Card, Avatar, Badge, Tooltip
    • Charts: LineChart, BarChart, PieChart, AreaChart

    Theme System¶

    The UI library uses a customizable theme system that supports:

    • Color Modes: Light, dark, and high contrast
    • Color Scales: Primary, secondary, accent, neutral
    • Typography: Fonts, scales, and responsive adjustments
    • Spacing: Consistent spacing scales
    • Breakpoints: Standard responsive breakpoints
    • Animations: Motion library for consistent animations

    Usage Example¶

    import {
  Button,
  Input,
  Card,
  Stack,
  Heading,
  Text
} from '@maily/ui';

function LoginForm() {
  return (
    <Card variant="elevated" padding="lg">
      <Stack spacing="md">
        <Heading size="lg">Sign In</Heading>
        <Text color="muted">Enter your credentials to continue</Text>

        <Input
          label="Email"
          type="email"
          placeholder="you@example.com"
          required
        />

        <Input
          label="Password"
          type="password"
          placeholder="Your password"
          required
        />

        <Button variant="primary" size="lg" width="full">
          Sign In
        </Button>
      </Stack>
    </Card>
  );
}

    Accessibility¶

    All UI components are built with accessibility in mind:

    • ARIA Attributes: Properly implemented ARIA roles and attributes
    • Keyboard Navigation: Full keyboard support
    • Focus Management: Visible focus indicators and logical focus order
    • Screen Reader Support: Appropriate text alternatives and semantic markup
    • Reduced Motion: Support for users with motion sensitivity
    • Color Contrast: Meets WCAG AA standards for all text

    Hooks Library¶

    Package: @maily/hooks Repository: packages/hooks

    The Hooks Library provides reusable React hooks that encapsulate common frontend logic, making it easier to implement complex behaviors consistently across the application.

    Key Hooks¶

    • Data Fetching: useQuery, useMutation, useInfiniteQuery
    • Forms: useForm, useField, useFormValidation
    • UI: useMediaQuery, useToast, useModal, useDisclosure
    • Auth: useAuth, usePermissions, useRole
    • Features: useCampaign, useTemplate, useSubscribers
    • Utilities: useDebounce, useThrottle, useLocalStorage, useEventListener

    Usage Example¶

    import { useQuery, useMutation } from '@maily/hooks';
import { Campaign } from '@maily/types';

function CampaignList() {
  // Fetch campaigns with automatic loading states and error handling
  const {
    data: campaigns,
    isLoading,
    error
  } = useQuery<Campaign[]>('/api/campaigns');

  // Create a mutation function for deleting campaigns
  const {
    mutate: deleteCampaign,
    isLoading: isDeleting
  } = useMutation('/api/campaigns', {
    method: 'DELETE',
    onSuccess: () => {
      // Refresh campaigns after deletion
      refetchCampaigns();
    }
  });

  if (isLoading) return <Spinner />;
  if (error) return <ErrorMessage error={error} />;

  return (
    <Table>
      {/* Table implementation */}
    </Table>
  );
}

    State Management¶

    Package: @maily/state Repository: packages/state

    The State Management package provides tools for managing global and local state across the application. It includes both client and server state management solutions.

    Key Features¶

    • Global State: Global application state management
    • Server State: Server state with caching, background updates, and optimistic UI
    • Form State: Complex form state management
    • Persistence: State persistence to localStorage
    • Performance: Optimized rerenders and performance monitoring
    • Dev Tools: Development tools for debugging state changes

    Implementation Details¶

    The package uses a combination of technologies:

    • Zustand: For global UI state
    • React Query: For server state
    • Form Libraries: For form state
    • Immer: For immutable state updates
    • Local Storage: For state persistence

    Usage Example¶

    import { useAppStore } from '@maily/state';
import { Button } from '@maily/ui';

function ThemeToggle() {
  const { theme, setTheme } = useAppStore(state => ({
    theme: state.theme,
    setTheme: state.setTheme
  }));

  return (
    <Button
      variant="ghost"
      onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}
    >
      {theme === 'light' ? 'Dark Mode' : 'Light Mode'}
    </Button>
  );
}

    Feature Packages¶

    Email Editor¶

    Package: @maily/email-editor Repository: packages/email-editor

    The Email Editor package provides a comprehensive WYSIWYG editor for creating and editing email templates with drag-and-drop functionality.

    Key Features¶

    • Block-Based Editor: Drag-and-drop block-based editing
    • Responsive Design: Mobile and desktop preview modes
    • Templates: Pre-built templates and template management
    • Personalization: Dynamic personalization tags insertion
    • Media Management: Image and media library integration
    • Code View: HTML and CSS code editing mode
    • AI Integration: AI-assisted content generation

    Usage Example¶

    import { EmailEditor, useEditorState } from '@maily/email-editor';
import { Button, Stack } from '@maily/ui';

function EmailTemplateEditor({ templateId }) {
  const {
    editorState,
    setContent,
    saveTemplate,
    isLoading
  } = useEditorState(templateId);

  return (
    <Stack spacing="md">
      <div className="editor-container">
  <EmailEditor
          value={editorState.content}
          onChange={setContent}
          mediaLibrary={true}
          personalizationTokens={true}
          aiAssistant={true}
        />
      </div>

      <Button
        onClick={() => saveTemplate()}
        disabled={isLoading}
      >
        Save Template
      </Button>
    </Stack>
  );
}

    Utility Packages¶

    Validation & Forms¶

    Package: @maily/validation Repository: packages/validation

    The Validation & Forms package provides tools for form validation, schema definition, and form state management.

    Key Features¶

    • Schema Validation: Schema-based validation
    • Form Generation: Form generation from schemas
    • Error Handling: Consistent error handling and display
    • Field Types: Support for all common field types
    • Custom Validation: Custom validation rules
    • Conditional Logic: Conditional form logic

    Usage Example¶

    import { createSchema, useValidatedForm, Field } from '@maily/validation';
import { Button, Stack } from '@maily/ui';

// Define a schema for campaign creation
const campaignSchema = createSchema({
  name: {
    type: 'string',
    required: true,
    min: 3,
    max: 100
  },
  subject: {
    type: 'string',
    required: true,
    max: 150
  },
  fromEmail: {
    type: 'string',
    required: true,
    format: 'email'
  },
  scheduledFor: {
    type: 'date',
    required: false,
    min: 'now'
  }
});

function CampaignForm({ onSubmit }) {
  const {
    values,
    errors,
    handleChange,
    handleSubmit,
    isValid
  } = useValidatedForm({
    schema: campaignSchema,
    initialValues: {
      name: '',
      subject: '',
      fromEmail: '',
      scheduledFor: null
    }
  });

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <Stack spacing="md">
        <Field
          name="name"
          label="Campaign Name"
          value={values.name}
          onChange={handleChange}
          error={errors.name}
        />

        <Field
          name="subject"
          label="Email Subject"
          value={values.subject}
          onChange={handleChange}
          error={errors.subject}
        />

        <Field
          name="fromEmail"
          label="From Email"
          type="email"
          value={values.fromEmail}
          onChange={handleChange}
          error={errors.fromEmail}
        />

        <Field
          name="scheduledFor"
          label="Schedule For"
          type="datetime"
          value={values.scheduledFor}
          onChange={handleChange}
          error={errors.scheduledFor}
        />

        <Button type="submit" disabled={!isValid}>
          Create Campaign
        </Button>
      </Stack>
    </form>
  );
}

    Internationalization¶

    Package: @maily/i18n Repository: packages/i18n

    Date & Time¶

    Package: @maily/date-time Repository: packages/date-time

    The Date & Time package provides utilities for working with dates, times, and timezones consistently across the application.

    Key Features¶

    • Formatting: Consistent date and time formatting
    • Parsing: Date and time parsing from various formats
    • Timezone Handling: Timezone-aware date and time manipulation
    • Relative Times: Relative time calculations and formatting
    • Date Math: Date arithmetic and calculations
    • Calendar Utilities: Calendar-related utilities
    • Date Selection: Date selection and range selection utilities

    Usage Example¶

    ```tsx import { formatDate, parseDate, getRelativeTime, addDays } from '@maily/date-time';

    function DateDisplay({ isoString, timezone = 'America/New_York' }) { // Parse the ISO string const date = parseDate(isoString);

    // Format for display in the user's timezone const formatted = formatDate(date, { format: 'full', timezone });

    // Get the relative time (e.g., "2 days ago") const relative = getRelativeTime(date);

    // Calculate a date in the future const nextWeek = addDays(date, 7); const nextWeekFormatted = formatDate(nextWeek, { format: 'medium' });

     return (
<div>
  <div>Formatted: {formatted}</div>
  <div>Relative: {relative}</div>
  <div>Next week: {nextWeekFormatted}</div>
</div>

    ); } ```

    Development Workflow¶

    The development workflow for frontend packages follows these key principles:

    1. Local Development: Run npm run dev to start the development server
    2. Testing: Write unit tests for all components with npm run test
    3. Building: Build packages with npm run build
    4. Documentation: Document components in Storybook with npm run storybook
    5. Versioning: Use changesets for versioning with npm run changeset

    Dependencies & Versioning¶

    Frontend packages use a consistent set of dependencies:

    • React: v18.2.0 for UI components
    • TypeScript: v5.2.0 for type safety
    • TailwindCSS: v3.3.0 for styling
    • React Query: v5.0.0 for data fetching
    • Zustand: v4.4.0 for state management
    • Jest: v29.5.0 for testing

    All packages follow semantic versioning and use Changesets for version management.

    Contributing¶

    When contributing to frontend packages:

    1. Follow the code style guidelines
    2. Ensure all tests pass before submitting PR
    3. Document components and hooks with JSDoc comments
    4. Create Storybook stories for UI components
    5. Update the changelog using Changesets
