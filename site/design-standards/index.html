    Overview¶

    This document consolidates all design standards and guidelines for the Maily platform. It serves as the single source of truth for maintaining design consistency across all interfaces and experiences. These standards apply to the web application, marketing website, email templates, and all other customer-facing materials.

    Design Principles¶

    Our design is guided by these core principles:

    1. Clarity First: Prioritize clarity and readability in all interfaces. Information should be easily scannable and comprehensible.

    2. Purposeful Simplicity: Simplify without sacrificing functionality. Every element should have a clear purpose and provide value.

    3. Intuitive Workflows: Create natural, predictable flows that match user expectations and mental models.

    4. Consistent Patterns: Use consistent interaction patterns throughout to reduce cognitive load.

    5. Data-Informed: Design decisions should be supported by data and user research.

    6. Accessible by Default: Design for inclusivity from the start, not as an afterthought.

    7. Scalable Systems: Create design systems that scale across different screen sizes, use cases, and future growth.

    8. Delightful Details: Add thoughtful micro-interactions and visual refinements that enhance the experience without distracting from it.

    Brand Guidelines¶

    Logo¶

    The Maily logo consists of the wordmark, sometimes accompanied by the envelope symbol. The logo should always appear in its original proportions and should never be stretched, distorted, or modified.

    Logo Variations¶

    Variation Usage
    Full Color Primary usage on light backgrounds
    Reversed For dark backgrounds
    Monochrome When color isn't available
    Symbol Only For favicons, app icons, and small spaces

    Logo Clear Space¶

    Maintain a minimum clear space around the logo equal to the height of the "M" in the wordmark.

    Misuse¶

    • Do not change the logo colors
    • Do not add effects (shadows, gradients, etc.)
    • Do not rotate or skew
    • Do not alter proportions
    • Do not place on busy backgrounds without sufficient contrast

    Brand Voice¶

    Our written communication follows these guidelines:

    • Clear: Direct and straightforward, avoiding jargon
    • Helpful: Focusing on solutions and user benefits
    • Friendly: Conversational but professional
    • Confident: Authoritative without being pushy
    • Concise: Respecting users' time by being brief

    Brand Personality¶

    Maily's brand personality traits:

    • Trustworthy: Reliable, secure, and dependable
    • Knowledgeable: Expert in email marketing
    • Approachable: Easy to use and understand
    • Efficient: Saving time and reducing complexity
    • Innovative: Forward-thinking but pragmatic

    Typography¶

    Maily uses a carefully selected type system to ensure readability and hierarchy across all platforms.

    Font Families¶

    Primary Fonts¶

    • Headings: Inter (Sans-serif)
    • Body: Inter (Sans-serif)
    • Monospace: JetBrains Mono (for code examples)

    Fallback Stack¶

    font-family: Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen, Ubuntu, Cantarell, "Open Sans", "Helvetica Neue", sans-serif;

    Type Scale¶

    We use a modular scale with a ratio of 1.2 (minor third):

    Level Size (rem) Weight Line Height Usage
    H1 2.986rem (47.78px) 700 1.1 Page titles
    H2 2.488rem (39.81px) 700 1.1 Section headers
    H3 2.074rem (33.18px) 600 1.2 Subsection headers
    H4 1.728rem (27.65px) 600 1.2 Card titles
    H5 1.44rem (23.04px) 600 1.3 Minor headings
    H6 1.2rem (19.2px) 600 1.3 Small headings
    Body Large 1.125rem (18px) 400 1.5 Featured content
    Body 1rem (16px) 400 1.5 Default body text
    Body Small 0.875rem (14px) 400 1.5 Secondary text
    Caption 0.75rem (12px) 400 1.4 Labels, captions

    Font Weights¶

    • Regular (400): Body text, UI elements
    • Medium (500): Emphasis, active states
    • Semi-Bold (600): Secondary headings, important UI elements
    • Bold (700): Primary headings, strong emphasis

    Typographic Guidelines¶

    • Line Length: Aim for 60-75 characters per line for optimal readability
    • Alignment: Left-align text (except for specific UI cases)
    • Letter Spacing: Slight tightening (-0.015em) for headings, normal for body text
    • Paragraph Spacing: 1.5Ã— the text size (e.g., 24px spacing for 16px text)
    • Emphasis: Use font weight rather than italics for emphasis
    • Numbers: Use tabular figures for tables and data displays

    • Background Contrast: Maintain sufficient contrast between background layers
    • Color Meaning: Don't rely solely on color to convey meaning
    • Dark Mode: Design colors with dark mode adaptation in mind

    Layout & Spacing¶

    A consistent layout and spacing system helps create visual harmony and improve usability.

    Grid System¶

    We use a 12-column grid system with responsive breakpoints:

    xs < 576px 4 16px 16px

    Spacing Scale¶

    We use an 8px spacing scale:

    Token Value Usage
    --space-0 0px No space
    --space-1 4px Minimal spacing, icons
    --space-2 8px Tight spacing, compact elements
    --space-3 12px Form controls, small gaps
    --space-4 16px Standard spacing
    --space-5 24px Medium spacing
    --space-6 32px Large spacing
    --space-7 48px Section spacing
    --space-8 64px Large section spacing
    --space-9 96px Extra large section spacing
    --space-10 128px Maximum spacing

    Layout Principles¶

    • Consistent Spacing: Use the spacing scale for all layout measurements
    • Content Density: Balance information density with whitespace
    • Visual Hierarchy: Use size, spacing, and color to establish hierarchy
    • Container Width: Content containers should not exceed 1400px
    • Alignment: Maintain consistent alignment across related elements
    • Responsive Behavior: Layout should adapt gracefully across screen sizes

    Common Layout Patterns¶

    • Card Grid: Used for displaying collections of similar items
    • Two-Column: Primary content (70%) with sidebar (30%)
    • Split Screen: Equal split for comparison or contrasting content
    • Content + Sidebar: Main content with supporting information
    • Page Header: Title, description, and key actions
    • Dashboard Layout: Multiple cards arranged in a grid

    Components¶

    Our component system provides consistent, reusable UI elements across the application.

    Button System¶

    Button Variants¶

    Variant Usage Appearance
    Primary Main actions, primary flows Solid blue background, white text
    Secondary Alternative actions White background, blue border and text
    Tertiary Less important actions No background/border, blue text
    Danger Destructive actions Solid red background, white text
    Ghost Minimal visual presence No background/border until hover

    Button Sizes¶

    Size Height Padding Font Size Usage
    Small 32px 12px 14px Compact spaces, secondary actions
    Medium 40px 16px 16px Standard size for most uses
    Large 48px 20px 16px Primary page actions, CTAs

    Button States¶

    • Default: Normal appearance
    • Hover: Slightly darker background
    • Active: Darker background, slight scale reduction
    • Focus: Focus ring (2px) with slight offset
    • Disabled: Reduced opacity, non-interactive
    • Loading: Loading indicator replacing or next to label

    Iconography¶

    Our icon system uses consistent, simple icons that enhance usability and aesthetics.

    Icon Library¶

    Maily uses a custom icon library built on a 24Ã—24px grid with 1.5px stroke weight.

    Icon Categories¶

    • Navigation: Icons used in navigation elements
    • Actions: Icons representing user actions
    • Objects: Icons representing objects or items
    • Status: Icons indicating status or state

    Icon Guidelines¶

    • Consistency: Maintain visual consistency across all icons
    • Simplicity: Use simple, recognizable shapes
    • Alignment: Icons should align to the pixel grid
    • Size: Use consistent icon sizes (16px, 20px, 24px)
    • Color: Icons inherit text color by default

    • Standalone: Icons that function without labels (must be universally recognized)

    Email Design¶

    Guidelines for designing Maily's transactional and marketing emails.

    Email Layout¶

    • Maximum width of 600px
    • Single-column layout preferred
    • Responsive design adapting to mobile devices
    • Clear visual hierarchy

    Email Typography¶

    • Use web-safe fonts or fallbacks
    • Minimum font size of 14px for body text
    • Line height of 1.5 for readability
    • Limit to 2-3 font styles per email

    Email Colors¶

    • Use brand colors consistently
    • Ensure sufficient contrast for readability
    • Include fallback colors for email clients that don't support your palette
    • Avoid background images with text overlay

    Email Components¶

    • Header with logo
    • Hero section for main message
    • Content blocks with clear spacing
    • Call-to-action buttons (minimum height: 44px)
    • Footer with legal information and unsubscribe option

    Email Best Practices¶

    • Design for image blocking (use alt text, don't rely solely on images)
    • Create clear hierarchy with headings and white space
    • Optimize for mobile viewing
    • Keep file sizes small
    • Test across multiple email clients

    Accessibility¶

    Our design system is built with accessibility as a core principle, not an afterthought.

    Accessibility Standards¶

    We follow WCAG 2.1 AA standards as a minimum requirement:

    • Perceivable: Information must be presentable to users in ways they can perceive
    • Operable: User interface components must be operable
    • Understandable: Information and operation must be understandable
    • Robust: Content must be robust enough to work with assistive technologies

    Color & Contrast¶

    • Text meets contrast requirements (4.5:1 for normal text, 3:1 for large text)
    • Don't rely solely on color to convey information
    • Provide sufficient contrast between UI elements
    • Support high contrast mode

    Keyboard Navigation¶

    • All interactive elements are keyboard accessible
    • Focus states are clearly visible
    • Logical tab order follows visual layout
    • No keyboard traps
    • Provide keyboard shortcuts for power users

    Screen Readers¶

    • All images have appropriate alt text
    • Use semantic HTML elements
    • ARIA attributes supplement HTML when needed
    • Ensure dynamic content changes are announced
    • Test with common screen readers

    Other Considerations¶

    • Support text resizing up to 200% without loss of content
    • Ensure sufficient touch target size (minimum 44Ã—44px)
    • Provide alternatives for time-based media
    • Avoid content that flashes more than three times per second
    • Design forms with clear labels and error messages

    Design Process¶

    Our design process ensures consistent, high-quality experiences across the platform.

    Design Workflow¶

    1. Research & Discovery: Understand user needs and business requirements
    2. Ideation: Explore multiple approaches to solve the problem
    3. Wireframing: Create low-fidelity mockups to establish structure
    4. Visual Design: Apply visual design language to wireframes
    5. Prototyping: Create interactive prototypes for testing
    6. User Testing: Validate designs with real users
    7. Refinement: Iterate based on feedback
    8. Handoff: Prepare designs for development
    9. Implementation: Support during development
    10. Review: Evaluate the implemented design

    Design Critique¶

    Regular design reviews ensure quality and consistency:

    • Weekly team critiques
    • Structured feedback format
    • Focus on user goals and business objectives
    • Reference design principles and standards

    Design Tools¶

    • Figma: Primary design tool for UI/UX
    • Adobe Creative Suite: Brand assets and marketing materials
    • Maze: User testing and validation
    • Abstract: Version control for design files
    • Zeplin: Design handoff and collaboration

    Design-to-Development Handoff¶

    • Comprehensive component specifications
    • Interactive prototypes demonstrating behavior
    • Accessibility requirements
    • Responsive behavior documentation
    • Assets in appropriate formats
