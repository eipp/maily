    Overview¶

    The Maily backend ecosystem consists of a collection of specialized packages designed to provide core functionality, services, and utilities. These packages follow a modular design to promote code reuse, separation of concerns, and maintainable architecture.

    Our packages utilize a monorepo structure managed with Turborepo, allowing for individual versioning while maintaining cross-package compatibility. This document provides comprehensive documentation for all backend packages within the Maily platform.

    Package Architecture¶

    The backend packages follow a layered architecture:

    1. Core Layer: Foundational packages for database access, schema definitions, and server utilities
    2. Service Layer: Domain-specific services implementing business logic
    3. API Layer: GraphQL and REST API implementations
    4. Infrastructure Layer: Integration with cloud services and infrastructure components

    The dependency flow follows a strict hierarchy to prevent circular dependencies:

    Infrastructure Layer → API Layer → Service Layer → Core Layer

    Package configurations are standardized with consistent: - TypeScript configuration - ESLint and Prettier settings - Testing framework - Documentation approach - Deployment process

    Core Packages¶

    @maily/server-core¶

    Purpose: Core server utilities and middleware for Node.js applications.

    Key Features: - HTTP server abstractions - Middleware ecosystem - Error handling framework - Logging infrastructure - Configuration management - Rate limiting - Request validation - Performance monitoring - Health check system

    Key Components: - Server: Main server abstraction - Router: Request routing - Middleware: Common middleware collection - Logger: Structured logging - Config: Configuration management - ErrorHandler: Centralized error handling - Validator: Request validation - Metrics: Performance metrics collection - HealthCheck: System health monitoring

    Usage: 

    import { Server, Router, Logger, Middleware } from '@maily/server-core';

// Create a new server instance
const server = new Server({
  port: 3000,
  logger: new Logger({ level: 'info' }),
});

// Create a router
const router = new Router();

// Add routes
router.get('/hello', async (req, res) => {
  res.json({ message: 'Hello, World!' });
});

// Add middleware
server.use(Middleware.cors());
server.use(Middleware.bodyParser());
server.use(Middleware.requestId());
server.use(Middleware.errorHandler());

// Use the router
server.use('/api', router);

// Start the server
server.start();

    Technical Details: - Package size: 42.1 KB (gzipped) - Built on: Node.js HTTP module with custom abstractions - Performance optimized with streaming support - Compatible with Express middleware - Full TypeScript support

    @maily/database¶

    Purpose: Database access layer and ORM integration.

    Key Features: - Database connection management - Query building and execution - Migration system - Transaction support - Connection pooling - Query logging and performance monitoring - Data mapping and transformation - Soft deletion support - Pagination utilities

    Key Components: - Database: Main database connection manager - QueryBuilder: SQL query construction - Model: Base model for database entities - Migration: Database migration framework - Transaction: Transaction management - Repository: Data access patterns - Pagination: Pagination utilities - Mapper: Data transformation utilities

    Usage: 

    import { Database, Model } from '@maily/database';

// Initialize database connection
const db = new Database({
  host: 'localhost',
  port: 5432,
  username: 'postgres',
  password: 'password',
  database: 'maily',
});

// Define a model
class User extends Model {
  static tableName = 'users';
  static schema = {
    id: { type: 'uuid', primaryKey: true },
    email: { type: 'string', unique: true },
    name: { type: 'string' },
    createdAt: { type: 'timestamp', defaultValue: 'now()' },
    updatedAt: { type: 'timestamp', defaultValue: 'now()' },
  };
}

// Query the database
async function getUserById(id: string) {
  return await User.query().findById(id);
}

// Use transactions
async function createUserWithProfile(userData, profileData) {
  return await db.transaction(async (trx) => {
    const user = await User.query(trx).insert(userData);
    const profile = await Profile.query(trx).insert({
      ...profileData,
      userId: user.id,
    });
    return { user, profile };
  });
}

    Technical Details: - Package size: 78.3 KB (gzipped) - Built on: Drizzle ORM with custom extensions - Database support: PostgreSQL (primary), MySQL (secondary) - Performance optimized with prepared statements - Connection pooling with health monitoring

    @maily/schema¶

    Purpose: GraphQL and JSON schema definitions with validation.

    Key Features: - GraphQL schema definitions - JSON schema validation - Type generation - Schema documentation - Schema validation - Cross-schema references - Versioning support - Extension system

    Key Components: - GraphQLSchema: GraphQL schema builder - JsonSchema: JSON schema builder - Validator: Schema validation utilities - TypeGenerator: TypeScript type generation - SchemaRegistry: Schema versioning and registry - Documentation: Schema documentation generation - Extensions: Schema extension utilities

    Usage: 

    import { GraphQLSchema, JsonSchema } from '@maily/schema';

// Define a GraphQL schema
const userSchema = new GraphQLSchema(`
  type User {
    id: ID!
    email: String!
    name: String
    createdAt: DateTime!
    updatedAt: DateTime!
  }

  type Query {
    user(id: ID!): User
    users: [User!]!
  }

  type Mutation {
    createUser(email: String!, name: String): User!
    updateUser(id: ID!, name: String): User!
  }
`);

// Define a JSON schema for validation
const userInputSchema = new JsonSchema({
  type: 'object',
  required: ['email'],
  properties: {
    email: {
      type: 'string',
      format: 'email',
    },
    name: {
      type: 'string',
      minLength: 2,
    },
  },
});

// Validate data against the schema
const isValid = userInputSchema.validate({
  email: 'user@example.com',
  name: 'John Doe',
});

// Generate TypeScript types
userSchema.generateTypes('./src/types');

    Technical Details: - Package size: 56.7 KB (gzipped) - Built on: GraphQL.js and JSON Schema with custom extensions - Code generation with GraphQL Code Generator - Integration with TypeScript compiler API - Documentation generation with GraphQL Documentation

    @maily/auth¶

    Purpose: Authentication and authorization framework.

    Key Features: - User authentication - Role-based access control - Permission management - OAuth2 integration - JWT handling - Session management - Multi-factor authentication - Security auditing - Rate limiting

    Key Components: - Auth: Main authentication service - Permissions: Permission management - Roles: Role definitions and management - OAuth: OAuth provider integration - JWT: JWT token handling - Session: Session management - MFA: Multi-factor authentication - Audit: Security audit logging - RateLimit: Authentication rate limiting

    Usage: 

    import { Auth, Permissions } from '@maily/auth';

// Initialize authentication service
const auth = new Auth({
  jwtSecret: process.env.JWT_SECRET,
  expiresIn: '1d',
  refreshExpiresIn: '30d',
});

// Define permissions
const permissions = new Permissions({
  'campaigns:read': 'Read campaign data',
  'campaigns:write': 'Create and update campaigns',
  'campaigns:delete': 'Delete campaigns',
  'users:read': 'Read user data',
  'users:write': 'Create and update users',
  'users:delete': 'Delete users',
});

// Define roles with permissions
auth.defineRole('admin', [
  'campaigns:read', 'campaigns:write', 'campaigns:delete',
  'users:read', 'users:write', 'users:delete',
]);

auth.defineRole('editor', [
  'campaigns:read', 'campaigns:write',
  'users:read',
]);

auth.defineRole('viewer', [
  'campaigns:read',
  'users:read',
]);

// Authenticate a user
async function login(email, password) {
  const user = await auth.authenticate(email, password);

  if (user) {
    const token = auth.createToken(user);
    return { user, token };
  }

  throw new Error('Invalid credentials');
}

// Authorize an action
function canEditCampaign(user, campaignId) {
  return auth.hasPermission(user, 'campaigns:write');
}

    Technical Details: - Package size: 48.2 KB (gzipped) - Multiple authentication strategies: JWT, OAuth, API Key - Customizable password hashing with Argon2 - FIDO2/WebAuthn support for passwordless authentication - Integration with identity providers via OpenID Connect

    Service Packages¶

    @maily/email-service¶

    Purpose: Email delivery, templating, and management.

    Key Features: - Email delivery via multiple providers - Email templating engine - Attachment handling - Email verification - Bounce management - Delivery tracking - Email threading - Template validation - Spam score analysis

    Key Components: - EmailService: Main email service - Template: Email template management - Mailer: Email sending and delivery - Attachment: File attachment handling - Verification: Email address verification - BounceManager: Email bounce handling - DeliveryTracker: Email delivery tracking - SpamChecker: Email spam score analysis

    Usage: 

    import { EmailService, Template } from '@maily/email-service';

// Initialize email service
const emailService = new EmailService({
  defaultProvider: 'sendgrid',
  providers: {
    sendgrid: {
      apiKey: process.env.SENDGRID_API_KEY,
    },
    ses: {
      region: 'us-east-1',
      accessKeyId: process.env.AWS_ACCESS_KEY_ID,
      secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY,
    },
  },
});

// Create a template
const welcomeTemplate = new Template({
  name: 'welcome-email',
  subject: 'Welcome to {{companyName}}!',
  html: `
    <h1>Welcome, {{name}}!</h1>
    <p>Thank you for joining {{companyName}}. We're excited to have you!</p>
    <p>Click <a href="{{activationLink}}">here</a> to activate your account.</p>
  `,
  text: `
    Welcome, {{name}}!

    Thank you for joining {{companyName}}. We're excited to have you!

    Click here to activate your account: {{activationLink}}
  `,
});

// Register the template
emailService.registerTemplate(welcomeTemplate);

// Send an email using the template
async function sendWelcomeEmail(user) {
  await emailService.sendWithTemplate('welcome-email', {
    to: user.email,
    from: 'welcome@maily.example.com',
    templateData: {
      name: user.name,
      companyName: 'Maily',
      activationLink: `https://maily.example.com/activate/${user.activationToken}`,
    },
  });
}

    Technical Details: - Package size: 92.5 KB (gzipped) - Multiple provider support: SendGrid, Amazon SES, Mailgun, Postmark - Intelligent provider failover - Handlebars-based templating engine - MJML support for responsive email templates - Email analytics integration

    @maily/analytics-service¶

    Purpose: Data collection, processing, and reporting for analytics.

    Key Features: - Event collection and processing - Real-time analytics - Reporting and visualization - User behavior tracking - Funnel analysis - Cohort analysis - A/B testing - Goal tracking - Data export

    Key Components: - AnalyticsService: Main analytics service - EventCollector: Event data collection - Processor: Data processing pipeline - Reporter: Report generation - UserTracker: User behavior tracking - FunnelAnalyzer: Conversion funnel analysis - ABTester: A/B test management - GoalTracker: Goal completion tracking - Exporter: Data export utilities

    Usage: 

    import { AnalyticsService, EventCollector } from '@maily/analytics-service';

// Initialize analytics service
const analyticsService = new AnalyticsService({
  database: db,
  eventTTL: 90, // days
});

// Create an event collector
const eventCollector = new EventCollector({
  batchSize: 100,
  flushInterval: 60000, // 1 minute
});

// Track events
async function trackUserAction(userId, action, metadata = {}) {
  await eventCollector.track({
    userId,
    event: action,
    timestamp: new Date(),
    properties: metadata,
  });
}

// Define a funnel
analyticsService.defineFunnel('signup-completion', [
  'visit-signup-page',
  'start-signup-form',
  'submit-signup-form',
  'verify-email',
  'complete-profile',
]);

// Get funnel conversion rates
async function getSignupFunnelMetrics(startDate, endDate) {
  return await analyticsService.getFunnelMetrics('signup-completion', {
    startDate,
    endDate,
    segment: 'all-users',
  });
}

// Define an A/B test
analyticsService.defineABTest('homepage-cta', {
  variants: ['control', 'variant-a', 'variant-b'],
  metrics: ['click-through-rate', 'conversion-rate'],
  startDate: new Date('2025-01-01'),
  endDate: new Date('2025-01-31'),
});

// Get A/B test results
async function getABTestResults(testId) {
  return await analyticsService.getABTestResults(testId);
}

    Technical Details: - Package size: 108.4 KB (gzipped) - Built on: ClickHouse for analytics data storage - Real-time capabilities with WebSockets - Batch processing for high-volume events - Data aggregation with materialized views - Machine learning integration for predictive analytics

    Usage: 

    import { AIService, ContentGenerator, SubjectOptimizer } from '@maily/ai-service';

// Initialize AI service
const aiService = new AIService({

    anthropic: {
      apiKey: process.env.ANTHROPIC_API_KEY,

// Generate email content
const contentGenerator = new ContentGenerator({

async function generateEmailContent(topic, tone, length, audience) {
  return await contentGenerator.generate({
    topic,
    tone,

    baseSubject,
    audience,
    count: 5,
    includePredictedMetrics: true,

// Segment audience
async function segmentAudience(users, campaignType) {
  return await aiService.segmentAudience({
    users,
    campaignType,
    segmentCount: 3,
    includePredictedEngagement: true,
  });

    @maily/audience-service¶

    Purpose: User audience management and segmentation.

    Key Features: - Contact management - Audience segmentation - List management - Subscription management - Contact enrichment - Compliance management - Import/export capabilities - Segment analytics - Automation triggers

    Key Components: - AudienceService: Main audience service - ContactManager: Contact data management - Segmenter: Audience segmentation - ListManager: Distribution list management - SubscriptionManager: Subscription preferences - Enricher: Contact data enrichment - ComplianceManager: Regulatory compliance - Importer: Data import utilities - Exporter: Data export utilities - TriggerManager: Automation triggers

    Usage: 

    import { AudienceService, ContactManager, Segmenter } from '@maily/audience-service';

// Initialize audience service
const audienceService = new AudienceService({
  database: db,
  aiService,
});

// Manage contacts
const contactManager = new ContactManager({
  audienceService,
});

async function addContact(contactData) {
  // Validate and normalize the contact data
  const normalizedContact = await contactManager.normalize(contactData);

  // Add the contact
  return await contactManager.create(normalizedContact);
}

// Create segments
const segmenter = new Segmenter({
  audienceService,
});

async function createSegment(name, criteria) {
  return await segmenter.create({
    name,
    criteria,
    description: `Segment for ${name}`,
  });
}

// Find contacts in a segment
async function getContactsInSegment(segmentId, page = 1, limit = 50) {
  return await segmenter.getContacts(segmentId, { page, limit });
}

// Subscribe/unsubscribe contacts
async function updateSubscription(contactId, preferences) {
  return await audienceService.subscriptionManager.update(contactId, preferences);
}

// Import contacts from CSV
async function importContactsFromCSV(filePath, listId) {
  return await audienceService.importer.importFromCSV(filePath, {
    listId,
    deduplication: 'update',
    notifyContacts: true,
  });
}

    Technical Details: - Package size: 76.8 KB (gzipped) - Data storage: PostgreSQL with JSON column support - Compliance features for GDPR, CCPA, and CAN-SPAM - Real-time segment evaluation - Integration with third-party data providers - High-performance filtering for large audiences

    Development & Testing¶

    Testing Framework¶

    Our backend packages use a comprehensive testing approach:

    • Unit Tests: Testing individual functions and methods
    • Integration Tests: Testing interactions between components
    • API Tests: Testing the API endpoints
    • Performance Tests: Testing under load and stress conditions
    • Snapshot Tests: Testing for unexpected changes

    All packages maintain >90% test coverage, with critical path coverage at 100%.

    Testing tools: - Vitest for unit and integration tests - Supertest for API testing - k6 for performance testing - Cucumber for behavior-driven tests

    Benchmarking¶

    We maintain performance benchmarks for all critical operations:

    • Query performance
    • API response times
    • Memory usage
    • CPU utilization
    • Throughput under load

    Benchmarks are run as part of the CI pipeline to catch performance regressions.

    Mocking & Fixtures¶

    Development is supported by:

    • Mock service implementations for testing
    • Comprehensive fixture data for various scenarios
    • Factory functions for generating test data
    • Database seeding utilities
    • Mock API server for frontend development

    Usage Guidelines¶

    Getting Started¶

    To use Maily backend packages in a new service:

    1. Install the core packages: 

      pnpm add @maily/server-core @maily/database @maily/schema @maily/auth

    2. Set up a basic server: 

      import { Server, Middleware } from '@maily/server-core';
import { Database } from '@maily/database';
import { Auth } from '@maily/auth';

// Initialize database
const db = new Database({
  connectionString: process.env.DATABASE_URL,
});

// Initialize auth
const auth = new Auth({
  jwtSecret: process.env.JWT_SECRET,
  database: db,
});

// Create server
const server = new Server({
  port: process.env.PORT || 3000,
});

// Add middleware
server.use(Middleware.cors());
server.use(Middleware.bodyParser());
server.use(Middleware.requestId());
server.use(auth.middleware());

// Start server
server.start();

    3. Add service-specific packages as needed: 

      pnpm add @maily/email-service @maily/analytics-service

    Best Practices¶

    • Use dependency injection for testability
    • Follow repository pattern for data access
    • Implement error handling at service boundaries
    • Use transactions for multi-step operations
    • Implement proper logging at appropriate levels
    • Keep services stateless when possible
    • Use config management for environment-specific settings

    Contribution Guidelines¶

    Adding or Modifying Packages¶

    1. Discuss the new package in the RFC process
    2. Create a branch with the naming convention: feat/package-name/feature-name
    3. Implement the package with full TypeScript typing
    4. Add comprehensive tests (unit, integration, performance)
    5. Create documentation with usage examples
    6. Submit a pull request with the implementation

    Quality Standards¶

    All contributions must meet these standards: - TypeScript strict mode compliance - 90%+ test coverage - Performance benchmarking for critical operations - Documentation with examples - No security vulnerabilities - No deprecated APIs

    Versioning Strategy¶

    We follow semantic versioning for all packages:

    • Major version (x.0.0): Breaking changes that require migration
    • Minor version (0.x.0): New features in a backward-compatible manner
    • Patch version (0.0.x): Bug fixes and non-breaking changes

    Release schedule: - Patch releases: As needed for critical fixes - Minor releases: Bi-weekly with feature additions - Major releases: Quarterly with proper migration guides
