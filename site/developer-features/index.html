    API & Integration¶

    GraphQL API¶

    Maily provides a comprehensive GraphQL API for accessing and manipulating platform data:

    Schema Design¶

    • Well-documented schema with descriptions for all types and fields
    • Consistent naming conventions following GraphQL best practices
    • Versioning strategy with deprecation notices
    • Schema evolution principles to ensure backward compatibility
    • Comprehensive tooling for introspection and exploration

    Core Functionality¶

    • Query operations for retrieving data with precise field selection
    • Mutation operations for creating, updating, and deleting resources
    • Subscription operations for real-time updates
    • Batched operations for efficient bulk processing
    • Cursor-based pagination for list endpoints
    • Filtering, sorting, and search capabilities

    Performance Features¶

    • DataLoader implementation for batching and caching
    • Persisted queries for reduced network overhead
    • Cost analysis and query complexity limiting
    • Automatic query tracing and performance monitoring
    • Response caching with appropriate cache control

    Security¶

    • Fine-grained permission checks at the field level
    • Input validation using strong type checking
    • Rate limiting and throttling controls
    • Authentication with JWT and other methods
    • Detailed error handling with appropriate information exposure

    Developer Experience¶

    • GraphQL Playground for interactive exploration
    • Detailed documentation with examples
    • Schema change notifications
    • Field-level deprecation messaging
    • Integration with popular GraphQL clients

    REST API¶

    For systems that prefer REST architecture, Maily offers a REST API covering core functionality:

    Endpoint Structure¶

    • RESTful resource-based URL structure
    • Consistent versioning in URL path (/v1/, /v2/)
    • Predictable status code usage
    • Comprehensive error responses with error codes
    • HATEOAS links for resource discovery

    Core Functionality¶

    • CRUD operations for all primary resources
    • Filtering and pagination parameters
    • Bulk operation endpoints
    • Webhook management
    • Search endpoints with flexible query options

    Performance Features¶

    • ETag support for caching
    • Conditional requests (If-Modified-Since, If-None-Match)
    • Compression for large responses
    • Partial response using field selection parameters
    • Rate limit headers

    Security¶

    • OAuth 2.0 authentication
    • API key management
    • CORS configuration
    • Request signing for sensitive operations
    • IP allowlisting options

    Developer Experience¶

    • OpenAPI/Swagger documentation
    • Status dashboard with API health
    • Deprecation headers for evolving endpoints
    • Detailed API changelogs
    • SDK generation from OpenAPI spec

    Webhooks¶

    Maily provides webhooks for real-time event notifications:

    Event Types¶

    • Campaign events (sent, opened, clicked, bounced)
    • Contact events (created, updated, unsubscribed)
    • List events (created, updated, contact added/removed)
    • System events (quota reached, delivery issues)
    • Custom events from workflows
    • Compliance events (data export request, deletion request)

    Configuration¶

    • Webhook URL registration and verification
    • Event type selection
    • Batch settings for high-volume events
    • Retry configuration with backoff
    • Secret key for payload verification
    • Test event generation for development

    Payload Structure¶

    • Consistent event envelope format
    • Detailed event-specific data
    • Event timestamp and unique ID
    • Related resource IDs and URLs
    • Webhook subscription ID

    Security & Reliability¶

    • Signature verification using HMAC
    • IP allowlisting
    • TLS encryption requirement
    • Webhook logs and delivery status
    • Manual retrigger capabilities
    • Circuit breaker for failing endpoints

    Management¶

    • Webhook dashboard with delivery statistics
    • Event type filtering
    • Historical event viewer
    • Webhook debugging tools
    • Webhook analytics (response time, success rate)
