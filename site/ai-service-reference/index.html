    Overview¶

    The Maily AI Service provides comprehensive artificial intelligence capabilities to enhance the email marketing platform. It integrates with multiple AI providers (OpenAI, Anthropic, Google) through a flexible adapter architecture, enabling advanced content generation, optimization, analysis, and personalization features.

    This reference document consolidates all AI-related documentation, providing a comprehensive guide for developers working with the AI components of the Maily platform.

    Architecture¶

    Model Adapter Interface¶

    The AI capabilities are built on a flexible adapter pattern that allows for easy integration with different AI models:

    • Base Adapter Interface: Provides a common interface for all AI model implementations
    • Provider-Specific Adapters: Implement the base interface for specific providers (OpenAI, Anthropic, Google)
    • Factory Pattern: Manages adapter creation and allows dynamic switching between providers
    • Configuration Management: Supports customization of model parameters like temperature and token limits
    // Model adapter interface
interface ModelAdapter {
  // Core methods
  generateText(prompt: string, options?: GenerationOptions): Promise<string>;
  generateChat(messages: ChatMessage[], options?: ChatOptions): Promise<ChatResponse>;

  // Email-specific methods
  generateEmailSubject(content: string, options?: SubjectOptions): Promise<string[]>;
  analyzeEmailContent(content: string): Promise<AnalysisResult>;
  optimizeEmailContent(content: string, options?: OptimizationOptions): Promise<string>;
}

    Key Components¶

    1. ModelAdapter (Abstract Class)¶

    • Defines standard methods for text generation, chat completions, and email-specific operations
    • Provides configuration management for model parameters
    • Implemented by provider-specific adapters

    2. Provider Adapters¶

    • OpenAIAdapter: Implementation for OpenAI's GPT models
    • AnthropicAdapter: Implementation for Anthropic's Claude models
    • GoogleAIAdapter: Implementation for Google's Gemini models
    • Each adapter handles API communication, error handling, and response parsing

    3. AIService¶

    • Singleton service that provides a simplified interface to AI capabilities
    • Manages adapter selection and configuration
    • Exposes domain-specific methods for email marketing operations

    4. Enhanced Components¶

    • EnhancedAIService: Extends the base AIService with advanced capabilities
    • OctoToolsIntegration: Provides agent orchestration capabilities
    • LangfuseTracing: Provides tracing and observability
    • HeliconeIntegration: Provides cost tracking and optimization

    Features & Capabilities¶

    Email Content Generation¶

    The content generation capabilities allow users to create compelling email content with AI assistance:

    • Complete Email Generation: Create full email drafts based on campaign goals and audience
    • Section Generation: Generate specific sections of an email
    • Content Variations: Create multiple variants for A/B testing
    • Tone Adjustment: Modify content to match different brand voices and tones
    • Personalization: Generate dynamic content based on audience segments
    // Example generation options
interface ContentGenerationOptions {
  topic: string;
  audience: string;
  tone: 'professional' | 'casual' | 'friendly' | 'urgent';
  goal: 'inform' | 'convert' | 'engage' | 'educate';
  length: 'short' | 'medium' | 'long';
  includeHeadings: boolean;
  includeCTA: boolean;
}

// Generate content
const content = await aiService.generateEmailContent({
  topic: 'Summer Sale Announcement',
  audience: 'existing customers',
  tone: 'friendly',
  goal: 'convert',
  length: 'medium',
  includeHeadings: true,
  includeCTA: true
});

    Subject Line Optimization¶

    The subject line optimization features help users create more engaging subject lines:

    • Subject Generation: Create compelling subject lines based on email content
    • A/B Test Variations: Generate multiple subject line options for testing
    • Personalization: Insert personalization elements for higher engagement
    • Length Optimization: Adjust subject line length for different devices
    • Emoji Suggestions: Recommend appropriate emojis to increase open rates
    // Example usage
const subjectLines = await aiService.generateSubjectLines({
  emailContent: '...',
  variations: 5,
  includeEmojis: true,
  maxLength: 50
});

    Content Analysis¶

    The content analysis capabilities provide insights and recommendations for improving email content:

    • Readability Analysis: Evaluate and score content readability
    • Tone Analysis: Identify the tone and suggest improvements
    • Engagement Prediction: Estimate potential engagement based on content
    • Spam Trigger Detection: Identify potential spam triggers
    • Improvement Suggestions: Provide actionable recommendations
    // Example analysis result
interface ContentAnalysisResult {
  readabilityScore: number;
  tone: string;
  estimatedEngagement: {
    openRate: number;
    clickRate: number;
  };
  spamTriggers: string[];
  suggestions: {
    category: string;
    description: string;
    priority: 'high' | 'medium' | 'low';
  }[];
}

// Analyze content
const analysis = await aiService.analyzeEmailContent(emailContent);

    Template Recommendations¶

    The template recommendation system suggests the most appropriate templates based on campaign goals:

    • Goal-Based Recommendations: Suggest templates based on campaign objectives
    • Audience-Specific Suggestions: Recommend templates that work best for specific audience segments
    • Historical Performance: Consider past performance when recommending templates
    • Content Matching: Find templates that best match the campaign content

    Audience Segmentation¶

    The AI-powered audience segmentation helps create more targeted campaigns:

    • Behavioral Clustering: Group subscribers based on similar behaviors
    • Engagement Prediction: Predict which subscribers are most likely to engage
    • Optimal Send Time: Determine the best time to send to each segment
    • Content Affinity: Identify content preferences for each segment

    Performance Prediction¶

    The performance prediction capabilities help estimate campaign results before sending:

    • Open Rate Prediction: Estimate potential open rates
    • Click Rate Prediction: Predict click-through rates
    • Conversion Prediction: Estimate conversion potential
    • ROI Calculation: Project potential return on investment

    Image Generation¶

    The image generation capabilities enable creation of visual assets:

    • Email Banners: Create custom header images for campaigns
    • Product Visualizations: Generate product images from descriptions
    • Social Media Assets: Create coordinated social media graphics
    • Image Variations: Generate multiple versions of images for testing

    Frontend Integration¶

    All AI-related UI components follow these principles:

    • Client-side React components with 'use client' directive
    • Responsive design with mobile-first approach
    • Loading states and error handling
    • Toast notifications for user feedback
    • Accessible UI elements with keyboard navigation
    'use client';

import { useState } from 'react';
import { useAI } from '@maily/hooks/useAI';
import { Button, Textarea, Spinner, Toast } from '@maily/ui';

export function SubjectLineGenerator({ emailContent, onSelect }) {
  const [isGenerating, setIsGenerating] = useState(false);
  const [subjectLines, setSubjectLines] = useState([]);
  const { generateSubjectLines } = useAI();

  const handleGenerate = async () => {
    setIsGenerating(true);
    try {
      const lines = await generateSubjectLines(emailContent, { variations: 5 });
      setSubjectLines(lines);
    } catch (error) {
      Toast.error('Failed to generate subject lines');
      console.error(error);
    } finally {
      setIsGenerating(false);
    }
  };

  return (
    <div className="space-y-4">
      <div className="flex justify-between">
        <h3 className="text-lg font-medium">AI Subject Line Generator</h3>
        <Button
          onClick={handleGenerate}
          disabled={isGenerating}
          variant="primary"
        >
          {isGenerating ? <Spinner size="sm" /> : 'Generate'}
        </Button>
      </div>

      {subjectLines.length > 0 && (
        <ul className="space-y-2">
          {subjectLines.map((line, index) => (
            <li
              key={index}
              className="p-3 border rounded-md hover:bg-gray-50 cursor-pointer"
              onClick={() => onSelect(line)}
            >
              {line}
            </li>
          ))}
        </ul>
      )}
    </div>
  );
}

    Migration Guide¶

    Migrating from Legacy AI Package¶

    The legacy AI package (packages/ai) is deprecated and will be removed after December 31, 2025. To migrate to the enhanced service:

    1. Update Imports¶

    // Before
import { generateEmailContent } from '@maily/ai';

// After
import { emailContentService } from '@maily/services/ai';

    2. Update Function Calls¶

    // Before
const content = await generateEmailContent({
  topic: 'Welcome message',
  tone: 'friendly'
});

// After
const content = await emailContentService.generate({
  topic: 'Welcome message',
  tone: 'friendly',
  variations: 2 // New features available
});

    3. Add New Features¶

    The enhanced service provides many new capabilities not available in the legacy package:

    // New capabilities
const analysis = await emailContentService.analyze(content);
const optimized = await emailContentService.optimize(content);
const subjectLines = await emailContentService.generateSubjects(content, 5);

    Backend Usage¶

    // Server-side API route
import { AIService } from '@maily/services/ai';

export async function POST(request: Request) {
  const { content, options } = await request.json();

  try {
    const aiService = new AIService();

    Troubleshooting¶

    Common Issues¶

    Rate Limiting¶

    Symptoms: 429 status codes, delayed responses, failed requests Solutions: - Implement exponential backoff - Use request batching - Distribute requests across multiple keys - Monitor usage and set alerts

    Model Availability¶

    Symptoms: Service unavailable errors, model not found errors Solutions: - Configure fallback models - Monitor provider status - Implement circuit breakers - Add redundancy across providers

    Content Filtering¶

    Symptoms: Content rejected, policy violation errors Solutions: - Pre-filter user inputs - Adjust prompts to avoid sensitive topics - Implement content moderation - Provide clear guidelines to users

    Debugging Tools¶

    • AI Service Logs: Detailed logs of AI interactions
    • Request/Response Inspector: View full API requests and responses
    • Token Counter: Estimate token usage before API calls
    • Prompt Explorer: Test prompts with different configurations
    • Cost Calculator: Estimate API costs for different models
