    Maily Frontend Architecture Modernization¶

    Next.js App Router Implementation¶

    React Server Components (RSC)¶

    • Use React Server Components for data fetching components
    • Implement proper component segmentation (Server vs Client components)
    • Apply proper patterns for Server/Client components
    • Follow the "move down" principle for client components

    Server Actions¶

    • Implement form handling with Server Actions
    • Use Server Actions for mutations in data
    • Implement proper validation with Zod
    • Apply optimistic updates where appropriate

    Streaming and Suspense¶

    • Implement streaming for initial page loads
    • Use loading.tsx for automatic suspense boundaries
    • Create custom suspense boundaries for complex UIs
    • Implement skeleton loaders for suspended components

    Parallel Routes¶

    • Use parallel routes for dashboard layouts
    • Implement parallel routes for multi-step forms
    • Use @modal for modal dialogs in routes
    • Maintain state across parallel routes

    UI Framework Standardization¶

    Tailwind CSS and shadcn/ui¶

    • Create design tokens based on Tailwind config
    • Implement shadcn/ui components to replace Material UI
    • Create a component library based on shadcn/ui
    • Document component usage and patterns

    Theme Customization¶

    • Implement CSS Variables for theme values
    • Create dark/light mode toggles
    • Support system theme preferences
    • Implement custom color schemes

    Modern State Management¶

    Server State with TanStack Query¶

    • Implement TanStack Query for server state
    • Create custom hooks for common queries
    • Implement proper mutation patterns
    • Use optimistic updates for better UX

    Client State with Zustand¶

    • Create Zustand stores for global UI state
    • Implement proper store segmentation
    • Use middleware for persistence where needed
    • Create typed stores with TypeScript

    Form State¶

    • Use React Hook Form with Zod validation
    • Create reusable form components
    • Implement field-level validation
    • Provide form error handling patterns

    Implementation Guidelines¶

    Migration Strategy¶

    1. Start with shared components migration
    2. Migrate API routes to Route Handlers
    3. Implement RSC data fetching
    4. Migrate forms to Server Actions
    5. Implement streaming and suspense
    6. Add parallel routes for complex pages

    Performance Optimization¶

    • Implement component-level code splitting
    • Use dynamic imports for client components
    • Implement proper image optimization
    • Ensure proper font loading strategies

    Accessibility¶

    • Ensure all components meet WCAG AA standards
    • Implement proper keyboard navigation
    • Maintain focus management in dynamic UIs
    • Test with screen readers

    Code Quality¶

    • Use TypeScript strict mode
    • Implement comprehensive unit tests
    • Use Playwright for E2E testing
    • Implement proper CI/CD for frontend code
